# QIIME2 Workflow

## Responsibilities
This skill handles QIIME2 CLI workflows for data import, denoising, classifier use, phylogeny construction, alpha and beta diversity, and optional export to downstream tools. It consumes staged inputs under `/job/stage/microbiome-qiime2-workflow/` and optional `/job/stage/metadata.tsv`, and it writes reproducible results to `/job/analysis/microbiome-qiime2-workflow/` with generated code under `/job/reproducibility/generated_scripts/`.

Use it when provenance matters, when a CLI workflow is preferred, or when the user wants a QIIME2-native analysis. Do not use it to invent artifact contents, hide invalid manifests, or omit provenance-sensitive decisions such as sampling depth or classifier choice.

## Companion Documents

## Input Conventions
This skill expects the following layout under `/job/stage/`:

```text
/job/stage/
  microbiome-qiime2-workflow/
    manifest.tsv # required or equivalent import table
    data/        # optional raw FASTQ or exported artifacts
    options.yaml # optional truncation, classifier, sampling depth, or plugin hints
    notes.txt    # optional study notes
/job/stage/metadata.tsv  # optional, read if present
```

If a manifest is present, read it first and use it to locate the actual files. A valid manifest or equivalent import specification is required. If taxonomy assignment is requested, the classifier or database choice must be present. If any required input is missing, unreadable, or contradictory, report the problem and halt; do not fabricate artifacts, metadata, or provenance.

If optional metadata is absent, proceed without it and keep provenance-sensitive decisions explicit.

## Output Conventions
Write all results to `/job/analysis/microbiome-qiime2-workflow/`:

- `results/rep-seqs.qza`
- `results/taxonomy.qza`

- `visualizations/demux.qzv`
- `visualizations/taxa-barplot.qzv`
- `visualizations/diversity-results/`
- `reports/qiime2_summary.md`
- `microbiome-qiime2-workflow_meta.json`

Write reproducibility artifacts to `/job/reproducibility/`:

- `generated_scripts/microbiome-qiime2-workflow_<YYYYMMDD-HHMMSS>.sh`
- `logs/microbiome-qiime2-workflow.log`

The metadata file must record package versions, input paths, output paths, plugin and classifier choices, sampling depth, provenance-sensitive decisions, and a brief explanation of the workflow path. If the run halts, write the metadata file when possible and record the blocker.

## Workflow
### Step 1 — Read the request
Determine whether the user wants import only, denoising, taxonomy, diversity, or a full workflow. If the request is ambiguous, ask for the minimum needed details before choosing plugins or parameters.

### Step 2 — Run the workflow
Import data, denoise with DADA2 or Deblur as appropriate, assign taxonomy, build a tree when needed, and run the requested diversity analyses while preserving provenance.

### Step 3 — Report and archive
Summarize the chosen workflow path, sampling depth, and outputs, then save the script and metadata so another analyst can reproduce the provenance-tracked analysis.

## Strict Rules / 严格规则
1. Never hide manifest or metadata problems.
2. Preserve provenance-sensitive decisions such as sampling depth and classifier choice.
3. Do not claim outputs that were not generated by QIIME2.
4. Do not fabricate diversity or taxonomy results.
5. Distinguish DADA2 and Deblur workflow branches when relevant.
6. Keep provenance and input-validation language explicit in the archived script.

## Execution Model (Two-Stage) / 执行模型（两阶段）
This skill follows the two-stage downstream pattern when a reference script is available.

Two paths are strictly separated; mixing them will fail:

| Path | Permission | Purpose |
|---|---|---|
| `/pipeline/scripts/reference_microbiome-qiime2-workflow.sh` | ro | repository-published template, mounted ro by `prepare_downstream.sh`; do not modify |
| `/job/reproducibility/generated_scripts/microbiome-qiime2-workflow_<YYYYMMDD-HHMMSS>.sh` | rw | job-specific customized copy; `run_downstream.sh` executes the most recent one |

Standard flow:

1. `bash gateway/attach.sh --shell` to enter the already-running container (or `docker exec -it $(cat /data/output/<job-id>/.container_name) bash`).
2. Inspect `/job/stage/` with QIIME2 tooling to confirm the import specification, classifier choice, and sampling depth.
3. Copy the reference script into `/job/reproducibility/generated_scripts/`.
4. Edit the copy with the current job's plugin selection and provenance-sensitive parameters.
5. Run the job-specific script with the documented runtime command and archive the log.

**Do not** run the reference script directly; it bypasses customization and metadata archiving. **Do not** write generated scripts under `Skills/microbiome-qiime2-workflow/scripts/`; that directory is for templates, not job artifacts.
